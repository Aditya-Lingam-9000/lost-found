  TECHNICAL ARCHITECTURE
═══════════════════════════════════════════════════════════════════════════════

ALGORITHM COMPONENTS (6 techniques blended together):

1. Token-Set Fuzzy Matching (50% weight) ← MAIN WORKHORSE
   - RapidFuzz's token_set_ratio()
   - Handles: word reordering + typos + substrings in one formula
   - Best for real-world variations

2. Partial Ratio Matching (20% weight)
   - RapidFuzz's partial_ratio()
   - Substring matching in descriptions
   - Handles: "Pro" in "Pro Max"

3. Token-Level Fuzzy (15% weight)
   - Fuzzy match important (3+ char) words individually
   - Picks best match for each token
   - Granular comparison

4. Jaccard Similarity (10% weight)
   - Set-based intersection/union of tokens
   - Prevents false positives
   - Ensures actual word overlap

5. Semantic Grouping (5% weight)
   - Bonus if both in same category (phones, watches, locations, etc.)
   - Weak bonus to prevent mismatches between categories

6. Phonetic Matching (Bonus, max 0.15)
   - Levenshtein normalized distance for typo tolerance
   - Handles nickname/spelling variations

BLENDING FORMULA:
  final_score = (0.50 * token_set)
              + (0.20 * partial)
              + (0.15 * token_fuzzy)
              + (0.10 * jaccard)
              + (0.05 * semantic)
              + phonetic_bonus
